Linux Process run time (del interrupts)

首先, 来看时钟中断, 开启高精度定时系统后, 时钟中断的服务例程指向了 hrtimer_interrupt(), 并且设置为 one-shot() 模式, 之后用高精度定时器模拟时钟中断, 此时,真正的时钟中断服务例程为 tick_sched_timer(), 所以我们首先来先减去程序运行期间的时钟中断的时间.

首先是测试程序:

之后用 kprobe 探测读开始时间

这里kprobe在vfs_read 执行时,并且读文件与我们设定的文件一致的时候才去输出,这里有几点需要注意,第一是我们只需要探测第一次读的开始时间,我们把这个时间定为程序执行时间的开始,只输出一次,因为会有很多次对该文件的读,所以加个判定,另外用kretprobe可以探测程序的执行时长, kretprobe 可以探测到程序的执行开始以及结束, 但是我用kretprobe每次都会死机,暂时找不出代码哪里有问题,所以就用 kprobe + gettimeofday 去替代, 这两个可以得到程序的执行开头以及 退出时的时间.

那么接下来要如何减去这期间的时钟中断时长,我在时钟中断的执行开始加上一个标志位去判断是否输出时钟中断的进入以及退出时间,并将这个标志位从内核导出, 然后写一个内核模块去修改 这个标志位, 问题来了, 现在的时钟中断是个定时器, 在内核里找到执行它的代码是不易的,这里我们可以使用kretprobe 去探测时钟中断的时长, 标志位的方法可用于监测别的中断,
